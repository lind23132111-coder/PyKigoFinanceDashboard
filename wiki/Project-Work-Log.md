## 🛠 詳細開發日誌

### 🔹 Milestone V1.0: 專案基石與核心邏輯 (Core Foundation)
*   **開發階段 (Phase 1-6)**: 涵蓋專案從 0 到 1 的建立。
*   **重點實作內容**:
    *   **基礎建設**: 建立 Supabase 星狀結構、GitHub Actions 自動化市場報價同步。
    *   **核心功能**: 實作財務目標管理、資產權重映射、多幣別匯率轉換。
    *   **AI 戰情室**: 導入 Gemini API 提供雙向回饋的財務洞察。
    *   **安全性與分流**: 實作全站密碼保護、`NEXT_PUBLIC_DEMO_MODE` 雙版本部署。
*   **關鍵 Debug**: 修復 RLS 權限導致的靜默失敗、解決 JS 浮點數金額加總誤差。
*   **Lesson Learnt**: 在專案初期建立好「Demo 模式」與「自動化同步」是長遠維運的關鍵，這讓隱私保護與功能驗證能並行。

### 🔹 Milestone V1.1: 介面打磨與數據健壯 (Polish & Integrity)
*   **開發階段 (Phase 8-9)**: 優化跨平台體驗與系統穩定度.
*   **實作亮點**:
    *   **行動版深度優化**: 實作持久型底部導覽、滾動置頂篩選橫幅，重構 Wizard 觸控佈局。
    *   **代號搜尋進化**: 實作 Fetch 直連 Yahoo 搜尋，新增台股 `.TW` 自動補齊與全形字處理。
    *   **嚴格數據防護**: 實作 Demo 模式「禁止寫入」安全鎖與測試關鍵字警告，徹底杜絕數據污染。
    *   **環境整潔**: 清理 Supabase 中所有孤立的測試紀錄，並開立「環境分離」技術改進單。
*   **Lesson Learnt**: 行動版不只是縮小畫面，而是重新設計觸控動線；數據完整性防禦與功能開發同樣重要。

### 🔹 Milestone V1.2: 投資策略與深度分析 (Strategy & Deep Analysis)
*   **開發階段 (Phase 7 & 10)**: 從「監控」進化為「戰略指揮」。
*   **實作亮點**:
    *   **TradingView 整合**: 在 `/planning` 頁面嵌入專業 K 線圖，支援個股即時技術分析。
    *   **交易策略筆記**: 實作 `strategy_notes` 持久化儲存，支援紀錄個股目標買賣價與心得。
    *   **指揮中心介面**: 採用 Premium Glassmorphism 風格，整合持股選單、動態行情與戰術筆記面板。
*   **Lesson Learnt**: 將專業分析工具 (TradingView) 與私密筆記結合，能極大化系統對投資決策的價值。

### 🔹 Milestone V1.3: 響應式佈局演進 (Responsive Layout Evolution)
*   **開發階段 (Phase 11-12)**: 針對跨裝置體驗進行深度重構。
*   **實作亮點**:
    *   **雙端混合佈局**: 電腦版維持專業三欄式佈局，手機版改為導航列下拉選單與垂直堆疊以釋放寬度。
    *   **手機版體驗極大化**: 線圖高度拉升至 `600px` 以解決小螢幕顯示壓迫，並將戰術筆記改為常置式底部顯示。
    *   **代碼精簡**: 移除多餘的 `isNotesOpen` 狀態與連動開關，簡化手機端操作路徑。
*   **Lesson Learnt**: 佈局不應只是「隱藏或縮小」，而是應根據螢幕使用場景 (Context) 決定資料呈現的優先順序與常置性。

### 🔹 Milestone Maintenance: 環境分離與架構重構 (Env Separation & Refactor)
*   **開發階段 (Phase 13)**: 強化開發安全性與代碼維護性。
*   **實作亮點**:
    *   **環境分離工程**: 實作 `.env.local` 導向開發用 Supabase 專案，徹底保護正式數據不受開發測試影響。
    *   **架構整理**: 將 SQL 初始化與 Mock 數據腳本遷移至 `supabase/scripts/`，移除 redundant 檔案與空目錄。
    *   **統一客戶端**: 整合 `src/lib` 中的 Supabase Client，並全面更新專案匯入路徑，提升代碼健壯性。
*   **Lesson Learnt**: 及時的技術債清償 (Refactoring) 與嚴謹的環境隔離，是專案從小型 side project 邁向穩定產品的必經過程。

### 🔹 Milestone V2.0: 自動化記帳與 分帳結算 (AI Accounting & Split)
*   **開發階段 (2026/03)**: 針對家庭日常支出與對帳痛點進行開發。
*   **實作亮點**:
    - **AI Inbox 智慧核對**: 支援多種載具格式，實作 Gemini 智慧去重與 2x2 高效排版。
    - **批次快速設定**: 實作 Smart Input 頂部批次同步歸屬功能，解決多筆重複點擊痛點。
    - **全歷史明細彈窗**: 實作全螢幕互動彈窗，支援跨月關鍵字搜尋與「開始/結束」日期篩選。
    - **即時反饋機制**: 修復確認支出後明細消失的問題，實作 session-based 篩選穿透。
    - **分帳結算核心**: 實作負債對沖公式，支援「Both」歸屬選項以應對共同資產支出。
*   **Lesson Learnt**: 家庭記帳的核心不在於「詳盡」而在於「核對的速度」與「歷史回溯的精確度」。AI 佈局與時間過濾器的組合大幅提升了系統的專業感。

### 🔹 Milestone V2.2: 靈活過濾與 UI 精煉 (Filter & UI Refinement)
*   **開發階段 (2026/03)**: 進一步強化支出統計的維度與細節體驗。
*   **實作亮點**:
    - **靈活過濾模式**: 實作「月/季/年」三段式過濾，支援動態計算日均值與週期比較 (Period comparison)。
    - **明細彈窗深度拋光**: 解決手機版關閉鍵換行問題，大幅提升字體級別（+2pt），優化行動端閱讀感。
    - **自動化操作優化**: 在批次模式下實作「編輯即勾選」，節省使用者 30% 以上的點擊次數。
    - **行動佈局修復**: 修正底部導航列在部分裝置上的排版溢出，確保 7 項功能完美對齊。
    - **系統穩定度修復**: 完成 V2.2.9 穩定度與死鎖保護。
*   **Lesson Learnt**: UI 的「易讀性」在行動端是第一優先順序，而減少使用者的「點擊深度」則是提升黏著度的關鍵。

### 🔹 Milestone V2.2.9: 系統穩定度與報表強化 (Stability & Report Sync)
*   **開發階段 (2026/03)**: 修復持續性的 UI 掛起與環境死鎖問題，並強化歷史報表選取邏輯。
*   **實作亮點**:
    - **掛起保護機制**: 在 `useExpenses` 核心 Hook 中實作 10 秒安全超時，防止因數據載入中斷導致的無限轉圈圈。
    - **報表期數選取器**: 在 `Report` 頁面實作 Snapshot Selector，支援直接選取過往結算的紀錄點，並同步顯示「快照精確時間」。
    - **結算邏輯健壯化**: 為 `getSplitSettlement` 伺服器動作加入嚴格的 Null Guard，防止空數據導致的渲染崩潰。
    - **彈窗日期同步**: 修正 `AllExpensesModal` 初始化時未繼承父組件日期過濾條件的 Bug。
    - **核心查詢優化**: 修改 `getReportData` 為 ID 基礎索引，提升報表切換速度。
*   **Lesson Learnt**: 對於複雜的非同步載入架構，必須具備「容錯轉場」與「超時退出」機制，以確保最基本的用户體驗不會因單點失敗而永久鎖死。
